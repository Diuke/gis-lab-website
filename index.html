World Population Maps Intercomparison and Validation

Go to WebGIS!

About the Project

This project is about Intercomparision and Validation of two World Population Maps.

World population maps provide global information, evidence-based analytics, and knowledge on human presence on the planet. The ultimate goal of these maps is to ensure that every person is mapped and counted within critical operations such as policy making and disasters response.

  • We used the data from two world population raster maps that are worldwide used: GHS-Population Grid and WorldPOP-Population Counts.
  • GHS-Population Grid (2015) is used and maintained by the European Union. It is a raster map with CRS WGS84 and resolution of 1km for each pixel of raster.
    This spatial raster dataset depicts the distribution of population, expressed as the number of people per cell.
  • WorldPOP-Population Count (2015). Is managed by various academic institutions. It is a raster layer with CRS WGS84 and resolution of 1km for each pixel.
    This dataset tells the population density per each pixel.
  • Our group was in charge of intercomparing and validate the portion of the GHS-POP tiles corresponding to some Pacific Ocean Islands, the southern-most part of South America, Part of South Africa and some Atlantic Ocean Islands for both of the Raster Maps.
  • INTERCOMPARISON:

    Intercomparison is a comparison of two distinct datasets to assess to which extent two of them share the same information.
    To make intercomparison, the maps must have the same extent, resolution, pixel classes or variable, and CRS.
    For our project we compared the GHS-POP and WorlPOP datasets, that share the same variable (population density), are from the same year (2015), and share the same pixel resolution (1km).
    We had to align them to perform the intercomparison so they both shared the same extent.
    For intercomparison no external data is necessary.

  • VALIDATION:

    Validation is a comparison of a map with a reference dataset with the aim to determine map accuracy.
    Typically, reference data are collected in a field (on-site) survey or in photo-interpretation of very-high-resolution satellite imagery or orthophoto.

    We validated our raster maps GHS-POP and WorldPOP by creating a reference dataset based on three scenarios:
    0 = GHS not populated but WorldPOP populated.
    1 = GHS and WorldPOP either both populated or both unpopulated.
    2 = GHS populated and WorldPOP not populated.
    We used Bing maps and Google Maps as high resolution satellite maps for photo-classification, and the QGIS ACaTaMa plugin for sampling the data and classifying the reference dataset as populated or not populated taking as criteria if there was at least one house on the raster cell.
    Finally, we used a python script that performed the accuracy and error matrix calculation for both GHS-POP and WorldPOP maps.

Tools and Services Used

QGIS

QGIS is a free and open-source cross-platform desktop geographic information system application that supports viewing, editing, and analysis of geospatial data. It also provides processing tools like GRASS GIS and GDAL that were used in this project.

QGIS Plugins

We used the QGIS plugin AcATaMa Accuracy Assessment of Thematic Maps and Quick Map Services to include satellite imagery from different basemaps - Google Satellite and Bing Satellite. We also used python for automating tasks and processing with the pyQGIS library.

Vue.js

Vue is a progressive framework for building user interfaces. The core library is focused on the view layer only, and is easy to pick up and integrate with other libraries or existing projects. We used it to provide awesome interactivity to the WebGIS map!

Open Layers

OpenLayers is an open-source JavaScript library for displaying map data in web browsers as slippy maps. It provides an API for building rich web-based geographic applications similar to Google Maps and Bing Maps

GeoServer

GeoServer is an open source software server written in Java that allows users to share and edit geospatial data. Designed for interoperability, it publishes data from any major spatial data source using open standards.

AWS

AWS (Amazon Web Services) is a secured cloud services platform that offers computing power, database storage, content delivery and various other functionalities. We used AWS S3 to host this website and an EC2 instance to host Geoserver!

Challenges Faced

  • Size of rasters - Processing time.

    As the Worldpop-PopulationCount and GHS-POP rasters were very big, QGIS took long times processing the rasters for some tasks.
    The resulting rasters are 240MB each, posing also a problem to Geoserver, reason why we had to upgrade our initial 1GB RAM instance to one with 2GB.
    OpenLayers and Geoserver handles the raster data optimally, so despite being big rasters, the loading times are very fast and the layer appears with lower resolution on bigger extends, while it appears with higher resolution at close distances.

  • Familiarity with the QGIS Software

    It was a new exposure for some members our group to the QGIS Software. At some steps it took additional efforts to learn about specific Plugins or Processing tools.

  • Handling of Data Types.

    For comparison we needed the layers to be in integer data type. The default QGIS raster calculator always convert the output layer to float, so we instead used the GRASS raster calculator, that is harder to use but is more flexible because it allows to pick the data type of the output layer.

  • Handling of Null Values.

    We had to align the null values of different layers for intercomparison. The default QGIS raster calculator posed a big issue handling the null values, while the GRASS raster calculator did all the processing without any errors.

  • Hosting of Geoserver.

    Geoserver is a Java-based OGC web server. To expose the processed layers on the internet, not only locally, we had to upload Geoserver to an AWS EC2 instance with Ubuntu on top of an Apache Tomcat webserver, with all configuration needed, including the FTP service to upload the layers.
