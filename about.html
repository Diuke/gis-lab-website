About Project

Our project includes two major steps Intercomparision and Validation of two World Population Maps.

World population maps provide global information, evidence-based analytics, and knowledge on human presence on the planet. The ultimate goal of these maps is to ensure that every person is mapped and counted within critical operations such as policy making and disasters response.

  • Data we used are the two world population maps that are GHS-Population Grid and WorldPOP-Population Counts.
  • GHS-Population Grid (2015) is used and merged by the European Union. It is a raster map with CRS WGS84 and resolution of 1km for each pixel of raster.
  • WorldPOP-Population Count (2015). Is managed by various academic institutions. It is also a raster layer with CRS WGS84 and resoltion of map is 1km for each pixel.
  • We have intercompared and validated the portion corresponding to some Pacific Ocean Islands, south part of South America, Part of South Africa and some Atlantic Ocean Islands in both the Raster Maps.
  • In InterComparision we have compared the GHS-POP and WorlPOP maps that have similar phenomenon such as same year of data (2015), same resoltion of pixels to access how much they are similar to each other. It helped us to know how much they are similar to each other. We dont need any external data for intercomparision.
  • In Validation we have validated our raster maps GHS-POP and WorldPOP by classifiying the maps based on three scenarios (1). Where GHS populated but WorldPOP not populated. (2). Where GHS not populated and WorldPOP populated. (3). Where GHS and WorldPOP either both populated or both unpopulated.We have used external base maps Like Bing maps and Google Maps from Open Layer Plugin for validation and using ACaTaMa plugin for sampling the data.

Tools and Services Used

QGIS

QGIS is a free and open-source cross-platform desktop geographic information system application that supports viewing, editing, and analysis of geospatial data. It also provides processing tools like GRASS GIS and GDAL that were used in this project.

QGIS Plugins

We used the QGIS plugin AcATaMa Accuracy Assessment of Thematic Maps and Quick Map Services to include satellite imagery from different basemaps - Google Satellite and Bing Satellite. We also used python for automating tasks and processing with the pyQGIS library.

Vue.js

Vue is a progressive framework for building user interfaces. The core library is focused on the view layer only, and is easy to pick up and integrate with other libraries or existing projects. We used it to provide awesome interactivity to the WebGIS map!

Open Layers

OpenLayers is an open-source JavaScript library for displaying map data in web browsers as slippy maps. It provides an API for building rich web-based geographic applications similar to Google Maps and Bing Maps

GeoServer

GeoServer is an open source software server written in Java that allows users to share and edit geospatial data. Designed for interoperability, it publishes data from any major spatial data source using open standards.

AWS

AWS (Amazon Web Services) is a secured cloud services platform that offers computing power, database storage, content delivery and various other functionalities. We used AWS S3 to host this website and an EC2 instance to host Geoserver!

Challenges Faced

  • Processing of the system slow down.

    As the Worldpop-PopulationCount data was very big the Qgis tool slow down and thats why each processing was taking time to process.

  • New to the QGIS Framework

    It was a new exposure for our group to the QGIS tool, at some steps it took additional efforts to learn about the specific Plugin or Processing tool.

  • Handling of Data Type for each Layer.

    We need to have the integer data type for each layer but due to the use of raster calculator each time we have to be carefully convert each layer to integer data type.

  • Handling Null Values for each layers.

    We have to align the Null values for different layers for intercomparison. After each conversion of data type and processing of layer we have to check the null values carefuly.
